<objective>
## Goal
Implement `cli.js` in full — `install`, `uninstall`, and `report` subcommands with real `settings.json` mutation and `--dry-run` support.

## Purpose
`cli.js` is the user-facing entry point: `npx pls-fix install` wires the hooks into Claude Code; `uninstall` removes them cleanly; `report` renders the session report on demand. Without this, the hook and report modules exist but can never be activated.

  <name>Task 1: Implement cli.js with all three subcommands</name>
  <files>cli.js</files>
  <action>
    Replace the entire stub body with a full implementation. Keep the shebang and 'use strict'.

    **Imports needed:** `fs`, `os`, `path` (built-ins only).

    **Config path resolution** (top of file, used by install + uninstall):
    ```js
    const configDir = process.env.CLAUDE_CONFIG_DIR || path.join(os.homedir(), '.claude');
    const configFile = path.join(configDir, 'settings.json');
    ```

    **--dry-run detection:**
    ```js
    const DRY_RUN = args.includes('--dry-run');
    ```

    **install subcommand:**
    1. Build absolute hook commands using `process.execPath` + `path.resolve(__dirname, ...)`:
       ```js
       const nodeExec = process.execPath;
       const hookPath  = path.resolve(__dirname, 'src/hook.js');
       const reportPath = path.resolve(__dirname, 'src/report.js');
       const hookCmd   = (DRY_RUN ? 'PLS_FIX_DRY_RUN=true ' : '') + `${nodeExec} ${hookPath}`;
       const reportCmd = `${nodeExec} ${reportPath}`;
       ```
    2. Read settings.json — if absent or invalid JSON, start from `{ hooks: {} }`.
    3. Ensure `settings.hooks.UserPromptSubmit` and `settings.hooks.SessionEnd` are arrays (create if missing).
    4. Check if already installed (any entry with `_pls-fix: true` in either array) — if so, print "pls-fix is already installed." and exit 0.
    5. Push one outer entry per hook type:
       ```js
       settings.hooks.UserPromptSubmit.push({
         matcher: '',
         hooks: [{ type: 'command', command: hookCmd }],
         '_pls-fix': true,
       });
       settings.hooks.SessionEnd.push({
         matcher: '',
         hooks: [{ type: 'command', command: reportCmd }],
         '_pls-fix': true,
       });
       ```
    6. Ensure `~/.pls-fix/` dir exists (for streak file); create with `fs.mkdirSync({ recursive: true })`.
    7. Write settings.json back: `JSON.stringify(settings, null, 2)`.
    8. Print confirmation:
       ```
       [pls-fix] Installed! Claude Code will now bless your fix/debug prompts.
       [pls-fix] Hook registered at: <configFile>
       [pls-fix] Dry-run mode: ON   (if --dry-run)
       [pls-fix] Restart Claude Code to activate.
       ```

    **uninstall subcommand:**
    1. Read settings.json — if absent, print "pls-fix not installed." and exit 0.
    2. Filter hook arrays:
       ```js
       ['UserPromptSubmit', 'SessionEnd'].forEach(evt => {
         if (Array.isArray(settings.hooks[evt])) {
           settings.hooks[evt] = settings.hooks[evt].filter(e => !e['_pls-fix']);
           if (settings.hooks[evt].length === 0) delete settings.hooks[evt];
         }
       });
       ```
    3. Write settings.json back.
    4. Delete session temp file if it exists:
       ```js
       const SESSION_FILE = path.join(os.tmpdir(), 'pls-fix-session.json');
       if (fs.existsSync(SESSION_FILE)) fs.unlinkSync(SESSION_FILE);
       ```
    5. Print farewell message, then call `require('./src/report').printReport()` for final lifetime stats.
       Print farewell BEFORE printReport so report appears last:
       ```
       [pls-fix] Uninstalled. Goodbye, and godspeed.
       ```

    **report subcommand:**
    1. Simply call `require('./src/report').printReport()` — that's the whole implementation.

    **default case:**
    Print `Usage: pls-fix <install|uninstall|report> [--dry-run]` and exit 0.

    **Error handling:** Wrap install/uninstall in try/catch — on error, print `[pls-fix] Error: <message>` to stderr and exit 1.

    Avoid: writing any temp or intermediate files. Avoid: spawning child processes. Avoid: reading src/session.js directly in cli.js (session cleanup uses os.tmpdir path directly).
