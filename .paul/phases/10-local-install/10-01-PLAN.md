<objective>
## Goal
Add `--local` flag to `install` and `uninstall` that targets `.claude/settings.json` in the current working directory instead of `~/.claude/settings.json`.

## Purpose
Users can scope pls-fix to a specific project (project-level Claude Code config) rather than all sessions (user-level). Both installs can coexist.

## Output
- `cli.js` — `--local` flag resolves `configDir` to `process.cwd()/.claude`; `COMMANDS_DIR` stays global (slash commands are user-level only)
- `README.md` — documents local install usage
</objective>

@.paul/PROJECT.md
@cli.js
@README.md

<acceptance_criteria>

## AC-1: Local config path
Given a user runs `pls-fix install --local`
When the command executes
Then hooks are written to `.claude/settings.json` in the current working directory, not `~/.claude/settings.json`

## AC-2: Uninstall respects --local
Given a user runs `pls-fix uninstall --local`
When the command executes
Then hooks are removed from `.claude/settings.json` in the current working directory

## AC-3: Global install unaffected
Given a user has a global install (no `--local` flag)
When they run `pls-fix install` or `pls-fix uninstall` without `--local`
Then behavior is identical to before — `~/.claude/settings.json` is targeted

## AC-4: Slash command files stay global
Given `pls-fix install --local` runs
When it writes slash command files
Then they are still written to `~/.claude/commands/` (user-level), not a local commands dir

## AC-5: Confirmation message reflects install scope
Given a local or global install completes
When the success message is printed
Then it includes the resolved config file path so the user knows which file was modified (already done via `configFile` in log — verify message still present for local path)

<task type="auto">
  <name>Task 1 — Update cli.js: resolve config path based on --local flag</name>
  <files>cli.js</files>
  <action>
    The `--local` flag is already parsed from `args`. Use it to select `configDir`:

    Replace the single `configDir` declaration with conditional logic:
    ```javascript
    const LOCAL = args.includes('--local');
    const configDir = LOCAL
      ? path.join(process.cwd(), '.claude')
      : (process.env.CLAUDE_CONFIG_DIR || path.join(os.homedir(), '.claude'));
    ```

    `configFile` and `writeConfig()` already use `configDir` — no other changes needed there.

    `COMMANDS_DIR` must stay user-level regardless of `--local`. It already resolves from `CLAUDE_CONFIG_DIR` or `~/.claude` — do not change it.

    In the `install` case, update the log line that prints the install scope:
    ```javascript
    console.log(`[pls-fix] Installed! (${LOCAL ? 'local' : 'global'}) Claude Code will now bless your fix/debug prompts.`);
    ```

    In the `uninstall` case, update the not-installed message to be scope-aware:
    ```javascript
    console.log(`[pls-fix] pls-fix not installed${LOCAL ? ' locally' : ''}.`);
    ```

    No other changes needed — `isAlreadyInstalled()`, `writeConfig()`, hook entry structure, and slash command management all work correctly with the resolved `configDir`/`configFile`.
  </action>
  <verify>
    Run under CLAUDE_CONFIG_DIR isolation (global) and a separate tmp dir (local):
    ```bash

    LOCAL_DIR=$(node -e "const os=require('os'),path=require('path'),fs=require('fs'); const d=path.join(os.tmpdir(),'pls-fix-local-'+Date.now()); fs.mkdirSync(d,{recursive:true}); console.log(d);")

    # Local install (--local, from LOCAL_DIR as cwd)

    # Confirm: $LOCAL_DIR/.claude/settings.json has hooks
    # Confirm: $GLOBAL_DIR/settings.json is unchanged (no local hooks added)
    ```
    Also confirm slash command files are written to GLOBAL_DIR/commands/ for both install types.
