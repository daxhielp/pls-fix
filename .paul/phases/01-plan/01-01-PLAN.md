<task type="auto">
  <name>Task 2: Write IMPLEMENTATION-PLAN.md</name>
  <files>IMPLEMENTATION-PLAN.md</files>
  <action>
    Write a comprehensive implementation plan document at the project root.
    Use confirmed hook type from Task 1 + decision checkpoint.

    Required sections:

    **1. File Structure**
    Exact file tree matching PRD:
    ```
    pls-fix/
    ├── package.json
    ├── cli.js
    ├── src/
    │   ├── hook.js
    │   ├── classifier.js
    │   ├── tiers.js
    │   ├── session.js
    │   └── report.js
    └── README.md
    ```
    For each file: what it exports, what it imports, its responsibility.

    **2. Hook Registration**
    - Which hook type (confirmed in checkpoint)
    - Exact settings.json path and mutation format
    - How install adds entries, how uninstall removes them
    - Whether project-level (.claude/settings.json) or user-level (~/.claude/settings.json) is modified

    **3. Hook I/O Protocol**
    - What stdin contains (JSON schema)
    - How to extract the user prompt text
    - What stdout must contain to pass the modified prompt through
    - What happens if the hook exits non-zero

    **4. Classifier Design**
    - Eligibility check: exact keyword list from PRD, logic (any match = eligible)
    - Desperation scoring: all 10 signal rules from PRD with point values
    - Score → tier mapping: exact boundaries (0-1, 2-3, 4-5, 6-8, 9+)
    - Return shape: `{ shouldInject, tier, score, signals }`
    - Edge cases: empty prompt, non-string input, context object shape

    **5. Session Storage**
    - Temp file path: `path.join(os.tmpdir(), 'pls-fix-session.json')`
    - Full JSON schema (matches PRD exactly)
    - Streak file decision: `~/.pls-fix/streak.json` (persistent across sessions)
    - When session is created vs read vs reset

    **6. Tier Phrase Format**
    - Exact export shape for tiers.js
    - Whether single string or array per tier
    - How hook.js reads it
    - Placeholder format for scaffold: `[TIER_N_PHRASE]`

    **7. npx Distribution**
    - package.json `bin` field value
    - Required fields for npx compatibility
    - Node.js version requirement
    - No build step (pure JS)

    **8. Dry-run Mode**
    - Env var: `PLS_FIX_DRY_RUN=true`
    - Exact stderr output format
    - Behavior: no injection, log to stderr

    Write this as a clean reference document, not prose. Use headers and bullet points.
    Phase 2 (Architect) reads this to scaffold. Do not leave ambiguity.
  </action>
